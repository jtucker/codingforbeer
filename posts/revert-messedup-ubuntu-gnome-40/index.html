    Revert Messed up Ubuntu install of Gnome 40

    I tend to run the latest and greatest dev version of Ubuntu on my laptop (Surface Book 2 currently) and wanted to check out the new Gnome 40 release because it looks great and I really want some of those sweet workspace improvements.

    So I checked out the first PPA I could find via Reddit and fired it up.

    And then I couldn’t boot into the system. It looked very similar to this:

    credits: Khayordey Phemmy via StackOverflow

    As mild panic set in, I figured there has to be a way to rollback these changes and sure enough, GooBingDuck led me to pretty simple solution.

    1. Boot like you normally would but select your Recovery kernel. Mine was in the Advanced menu item in Grub
    2. Enable Networking in the menu. I needed to be hardwired in to get a network connection which I was lucky to have. I’m not sure how you would do it via Wi-Fi
    3. Enter a root shell
    4. Install ppa-purge 
      apt-get install ppa-purge
    5. Remove the PPA 
      ppa-purge ppa:shemgp/gnome-40
    6. Reboot

    Once the reboot finished, I was back to where I was pre-Gnome 40 and back to waiting patiently.
